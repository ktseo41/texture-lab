# 텍스트 레이어 — 향후 확장 메모

현재 (v0.4): 멀티라인 내용 · 크기(%높이) · 위치 X/Y(%) · 색 · 불투명도.
블록은 textX/textY 중심에 놓이고 행은 왼쪽 정렬(마스트헤드 스타일) 고정.
커서(`_`) 토글은 debug 전용 노출(localhost 또는 `?debug=1`), 파라미터는 항상 동작.

아직 안 만든 것 (요청 시 추가):

- **정렬 옵션** — 행 정렬 left/center/right 선택. 지금은 left 고정.
- **박스 크기 / 자동 줄바꿈** — 텍스트 박스 폭(%)을 주고 넘치면 자동 개행 + 크기 축소(fit).
  지금은 명시적 줄바꿈만 지원, 캔버스보다 길면 잘리지 않고 밖으로 나감.
- **드래그 이동** — 캔버스에서 텍스트를 직접 끌어 위치 조정 (textX/textY 슬라이더 대체).
- **회전** — textAngle(°). 소스 단계 회전이라 할프톤 각도와 간섭 없음.
- **폰트 선택** — 지금은 Pretendard 800 고정. 폰트 패밀리/웨이트 select,
  커스텀 웹폰트 URL 입력. 폰트 로딩 시 fontGen 범프 재사용.
- **자간/행간** — letterSpacing(em), lineHeight 슬라이더. 지금은 -0.03em / 0.98 고정.
- **다중 레이어** — 텍스트 블록 여러 개 (배열 파라미터 필요 — URL 인코딩 스키마 확장 필요).

구현 위치: 렌더링은 `src/engine/source.js` 텍스트 블록, 파라미터는 `src/state/params.js`,
UI는 `src/ui/controls.js`의 `g-text` 그룹. 새 파라미터는 반드시 `sourceKey()`에 포함할 것
(소스 캐시 무효화). URL 상태는 DEFAULTS와 타입이 같으면 자동으로 통과된다.
